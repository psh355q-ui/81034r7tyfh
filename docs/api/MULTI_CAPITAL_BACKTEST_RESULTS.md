## 🔍 발견 사항

### 문제 1: Small Capital (₩10M)
**증상**: 모든 거래 거부 (15/15건)  
**원인**: 최소 주문 크기 $1,000  
**설명**: ₩10M의 10% = ₩1M ≈ $833 < $1,000

**해결**: 최소 자본 요구사항 문서화  
→ Constitution은 ₩100M 이상 권장

### 문제 2: Large Capital (₩1B) - 해결됨! ✅
**초기 증상**: 모든 거래 거부 (15/15건)  
**원인**: 최대 주문 크기 $50,000  
**설명**: ₩1B의 10% = ₩100M ≈ $83K > $50K

**해결**: Constitution 수정
```python
# trading_constraints.py
# Before: 모든 자본에 $50K 상한 적용
if order_value_usd > MAX_ORDER_SIZE_USD:
    violations.append(...)

# After: $100K+ 자본은 비율 제한만

    if order_value_usd > MAX_ORDER_SIZE_USD:
        violations.append(...)
```

**결과**: ₩1B도 정상 작동! (+1.65%)

## 💡 Constitution 개선

### 확장 가능한 규칙 (Scalable Rules)

**철학적 정당성**:
> "Constitution은 모든 자본 규모에서 작동해야 한다.  
> 소액 투자자는 절대 금액으로 보호하고,  
> 대형 투자자는 비율로 관리한다."

**개선 내용**:
1. **소형 자본 ($100K 미만)**:
   - 최소 주문: $1,000
   - 최대 주문: $50,000
   - 비율 제한: 10%

2. **대형 자본 ($100K 이상)**:
   - 최소 주문: $1,000
   - 최대 주문: 없음 (비율만)
   - 비율 제한: 10%
